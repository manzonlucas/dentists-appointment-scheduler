Todas las funciones de la API corren bien desde Postman cumpliendo su funcion (GET, POST, PUT, DELETE).
No todas las funciones de la API estan integradas en el front, solo post, get y delete.

API DOCUMENTATION:

- OdontologoController
POST /odontologos => sin propiedad "id" en json, CREA un nuevo odontologo.
PUT /odontologos => con propiedad "id" en json, MODIFICA un odontologo ya existente.
GET /odontologos => trae lista de todos los odontologos.
GET /odontologos/{id} => trae odontologos con ese id.
DELETE /odontologos/{id} => elimina odontologos con ese id.

- PacienteController
POST /pacientes => sin propiedad "id" en json, CREA un nuevo paciente.
PUT /pacientes => con propiedad "id" en json, MODIFICA un paciente ya existente.
GET /pacientes => trae lista de todos los paciente.
GET /pacientes/{id} => trae pacientes con ese id.
DELETE /pacientes/{id} => elimina pacientes con ese id.

- TurnoController
POST /turnos => CREA un turno y recibe lo siguiente:
{ 
paciente: {id: x}, 
odontologo: {id: x}
}
PUT /turnos => idem con propiedad "id" en json, MODIFICA un turno.
GET /turnos => trae lista de todos los turnos.
GET /turnos/{id} => trae turno con ese id.
DELETE /turnos/{id} => elimina turno con ese id.
